# campaign Implementation Plan (Track B, Plan 3)

> **For agentic workers:** REQUIRED SUB-SKILL: superpowers:subagent-driven-development or executing-plans.

**Goal:** Add a manifest-driven bulk-publish `campaign` (plan, fill, run) so a large backlog can be authored once, reviewed, and executed resumably, with one confirmation per campaign instead of per cell.

**Architecture:** New module `crier/campaign.py`: a YAML manifest at `<site_root>/.crier/campaigns/<name>.yaml` of (post x platform) cells; `plan_campaign` seeds it from a content scan plus the registry (only not-yet-published cells; short-form cells get an empty `rewrite` field); `run_campaign` executes each pending cell through `publish_one`, records the result, and writes status back (resumable). CLI `crier campaign plan|run` and MCP `crier_campaign_plan` / `crier_campaign_run` are thin wrappers. Dry-run by default; `--apply` publishes.

**Tech stack:** Python 3.10+, PyYAML, existing `publish_one`, registry, config, utils.

## Manifest schema

```yaml
campaign: spring-backlog
created: 2026-06-06T00:00:00Z
platforms: [devto, hashnode, bluesky, mastodon]
posts:
  - canonical_url: https://metafunctor.com/post/foo/
    file: content/post/2026-.../index.md
    title: "Foo"
    targets:
      devto:    {status: pending}              # long-form
      bluesky:  {status: pending, rewrite: ""} # short-form (fill before run)
```

Cell `status`: `pending` | `published` | `failed` | `needs_rewrite`. On run, published cells gain `url`; failed cells gain `error`.

## Interfaces (verified)

- `config.get_site_root() -> Path | None` (manifest dir = `get_site_root() or cwd` + `/.crier/campaigns`).
- `utils.find_content_files(explicit_path=None) -> list[Path]`.
- `converters.parse_markdown_file(path) -> Article` (`.canonical_url`, `.title`).
- `registry.is_published(canonical_url, platform) -> bool`; `record_publication(...)`; `record_failure(...)`.
- `config.is_short_form_platform(platform) -> bool`; `get_api_key`; `get_platform_mode`; `platforms.PLATFORMS`.
- `publishing.publish_one(file, platform, *, rewrite_content=None, ...) -> PublishOutcome` (lazy import in run).

## Task 2: run engine

- [ ] `run_campaign(name, *, apply=False) -> CampaignRunSummary`: load manifest (error if missing). For each post/target cell: skip `published` (resume); short-form with empty `rewrite` -> `needs_rewrite` (no publish); else dry-run counts `pending`, or with apply call `publish_one(file, platform, rewrite_content=rewrite or None)`, then record_publication on success (cell -> published + url) or record_failure on failure (cell -> failed + error). Save manifest only when apply. Dry-run makes no publishes and no writes.
- Tests: dry-run no publish/no save; apply publishes + records + writes status; resume skips published; short-form empty rewrite -> needs_rewrite (publish_one not called); failure path records failure + cell failed; missing manifest -> error. Commit `feat(campaign): resumable run engine`.
